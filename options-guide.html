Claude Code 下段メニューの見方

初めての方でもわかる、3つの選択肢の説明

🤖
①モデル（AI の種類）
例: Default / Sonnet / Opus

Claude の「賢さのレベル」を選ぶ項目です。
料理で例えると、使う食材のグレードのようなものです。

選び方の目安
Default（おすすめ）：ふだん使いに最適。迷ったらこれ！
Sonnet：バランスが良く、多くの作業に向いている
Opus：難しい問題に強い上位モデル
📁
②リポジトリ（作業するフォルダ）
例: my-project / homepage

Claude がコードを読み書きする「プロジェクトのフォルダ」を選びます。
引っ越しで例えると、どの部屋（プロジェクト）で作業するかを決めるイメージです。

ポイント
「自分が作っているアプリやWebサイトのフォルダ」を選べばOKです。
フォルダはGitHubという場所に保存されています。
🌿
③ブランチ（作業の流れ）
例: main / 機能追加用

同じフォルダの中で「どのバージョンで作業するか」を選びます。
ノートで例えると、本番用のページと下書き用のページを切り替えるイメージです。

選び方の目安
main：完成版・本番用のブランチ（そのまま公開されるもの）
それ以外：新機能のテスト中・下書き用のブランチ
💡 まとめ：3つをひとことで言うと… 🤖 モデル＝AIの賢さを選ぶ
📁 リポジトリ＝どのプロジェクトで作業するかを選ぶ
🌿 ブランチ＝本番用か下書き用かを選ぶ
